# 简约笔记 App 完整开发任务书（AI 专用）

## 一、项目现状与目标

你已经完成了一个 Android 基础版笔记 App（Kotlin + Supabase），实现了：
- 用户注册/登录（Supabase Auth）
- 笔记的增删改查（本地 SQLite 与云端 Supabase 同步，自动同步）
- 文件夹分类（可选）
- 搜索笔记
- 基础设置（主题跟随系统等）

现在需要在此基础上，为同一个 App 添加 **Pro 和 APEX 功能**。所有用户使用同一个 APK，根据登录后从 `user_profiles` 表中读取的 `plan` 字段（`free` / `pro` / `apex`）来动态显示/隐藏功能入口，以及控制云端资源的下载权限。

**重要原则**：
- 所有关键资源（官方主题包、社区主题文件）必须存储在 Supabase Storage 中，并通过 RLS 策略控制下载权限。
- 涉及敏感权限检查的操作（如上传社区主题、生成动画配置）必须通过 Supabase Edge Function 完成，不能在客户端直接判断。
- 客户端本地的 `plan` 判断只用于 UI 显示，不用于安全决策。

## 二、已建好的 Supabase 表结构（请 AI 参考）

### 1. `user_profiles`
```sql
id uuid PK references auth.users
plan text default 'free'  -- 'free', 'pro', 'apex'
is_lifetime_free boolean default false
created_at, updated_at
```

### 2. `official_themes`
```sql
id uuid PK
name text
description text
preview_url text   -- Storage 路径，如 'official_theme_assets/theme1/preview.jpg'
config_url text    -- Storage 路径，如 'official_theme_assets/theme1/config.json'
is_free boolean default false
sort_order int
created_at
```

### 3. `community_themes`
```sql
id uuid PK
user_id uuid references auth.users
name text
description text
preview_url text
config_url text
is_public boolean default false
download_count int default 0
created_at, updated_at
```

### 4. `user_themes`
```sql
id uuid PK
user_id uuid references auth.users
name text
config jsonb
created_at, updated_at
```

### 5. `notes`（已有，不再改动）

### Storage Buckets
- `official_theme_assets`：存放官方主题的预览图和 config.json
- `community_theme_assets`：存放社区主题的预览图和 config.json

**RLS 策略已配置**：
- 只有 Pro/APEX 用户可以下载 `official_theme_assets` 中的文件（免费主题除外，但客户端会判断 `is_free` 字段）。
- 只有 APEX 用户可以上传/下载 `community_theme_assets` 中的 JSON 文件，但预览图所有人可读。

## 三、需要实现的功能清单（按优先级排序）

### 模块 A：权限管理与基础工具（必须先做）

#### A1. 全局权限管理类 `FeatureGuard`
- **文件路径**：`com.jianyinote.utils.FeatureGuard.kt`
- **功能**：
  - `suspend fun getCurrentPlan(): String`：从 `user_profiles` 表实时获取当前用户的 plan（缓存 5 分钟，也可提供强制刷新）。
  - `suspend fun isProOrApex(): Boolean`
  - `suspend fun isApex(): Boolean`
  - `fun showUpgradeDialog(context: Context, featureName: String)`：弹出一个对话框，说明需要升级到 Pro/APEX，并提供两个按钮：“稍后”（关闭），“了解详情”（跳转到你指定的 URL，先用 `https://your-website.com/upgrade` 占位）。
- **注意**：这些方法必须为 `suspend` 或使用回调，因为需要网络查询。在 UI 层调用时，要在协程或 ViewModel 中执行。

#### A2. 当前用户 Plan 的实时观察者 `UserPlanObserver`
- **文件路径**：`com.jianyinote.auth.UserPlanObserver.kt`
- **功能**：
  - 使用 `StateFlow<String>` 暴露当前 plan。
  - 在 Application 类中初始化，登录成功后从网络拉取一次，并监听 `user_profiles` 表的变化（通过 Supabase Realtime 可选，或简单地在每次打开设置页时刷新）。
  - 提供一个 `refresh()` 方法供手动刷新。

#### A3. 主题应用全局管理器 `ThemeManager`
- **文件路径**：`com.jianyinote.theme.ThemeManager.kt`
- **功能**：
  - 单例，负责加载和保存当前主题配置（颜色、字体、动画等）。
  - 主题配置格式为 JSON，存储到 `DataStore`。
  - 提供 `applyTheme(context: Context, config: JSONObject)` 方法，动态修改 App 的 UI（使用 Compose 或传统 View？假设你用 XML + DayNight，可以重新设置 `Theme` 并重启 Activity；如果用 Compose，则通过 `CompositionLocal` 传递）。
  - 默认主题为系统默认（浅色/深色跟随系统）。

### 模块 B：Pro 功能 - 主题自定义（用户自己调颜色/背景/字体）

#### B1. 主题编辑器界面 `ThemeEditorActivity`
- **入口**：仅在 `plan == 'pro' or 'apex'` 时显示在设置页。如果权限不足，按钮隐藏或调用 `FeatureGuard.showUpgradeDialog`。
- **UI 组件**：
  - 颜色选择器：用于主色调、背景色（可用开源库如 `com.github.skydoves:colorpicker-compose`）。
  - 背景图片选择：从相册选择图片，或者使用纯色。
  - 字体大小滑动条（14-24sp）。
  - 字体类型下拉框（系统默认、衬线、等宽）。
  - 实时预览区域：显示笔记列表和编辑器样式的 Demo 卡片。
- **保存逻辑**：
  - 用户点击“保存”，生成一个 JSON 配置，存储到本地 `DataStore`，并立即应用到全局。
  - 同时，如果是 APEX 用户，可额外显示“保存到云端”按钮（见模块 E）。
  - **不自动上传到任何云端表**，仅本地使用。

### 模块 C：Pro 功能 - 官方主题商店（下载精美主题包）

#### C1. 主题商店界面 `OfficialThemeStoreActivity`
- **入口**：设置页中，仅 Pro/APEX 可见（免费用户看不到，或看到后点击提示升级）。
- **UI**：使用 `RecyclerView` 展示 `official_themes` 表中所有主题（包括免费主题，免费用户也可以下载）。
- **每个主题项显示**：预览图、名称、描述、一个“下载/应用”按钮。
- **下载与应用逻辑**：
  1. 点击按钮时，先检查本地是否已下载该主题（根据主题 id 存储标志）。
  2. 若未下载，调用 Supabase Storage 的下载方法，根据 `config_url` 下载 JSON 文件。**下载时使用 Supabase 客户端，会自动携带用户 JWT**。如果用户权限不足（免费用户尝试下载 `is_free=false` 的主题），Storage RLS 会返回 403，客户端捕获后提示“请升级到 Pro”。
  3. 下载成功后，解析 JSON，调用 `ThemeManager.applyTheme()` 应用主题，并保存到本地缓存（标记已下载）。
  4. 若已下载，直接应用。
- **注意**：预览图可以直接通过 `preview_url` 加载，不需要权限检查（因为预览图通常公开）。

### 模块 D：APEX 功能 - 小型动画编辑器（自定义转场动画）

#### D1. 动画编辑器界面 `AnimationEditorActivity`
- **入口**：仅在 `plan == 'apex'` 时显示在设置页。
- **功能**：用户可以调整以下参数，实时预览效果：
  - 入场动画类型（缩放、平移、淡入）
  - 动画时长（200-800ms 滑块）
  - 缓动曲线（线性、加速、减速、弹性）
- **预览**：点击“测试动画”按钮，模拟打开一个笔记页面，使用当前设置的动画。
- **保存配置**：生成一个 JSON 配置，存储到本地 `DataStore`。之后在打开笔记时应用此动画。
- **云端安全**：**关键**：生成动画配置的算法必须放在 Supabase Edge Function 中，而不是客户端本地。客户端只将参数发给 Edge Function，由 Edge Function 返回最终的动画配置 JSON。这样可以防止用户破解客户端直接获得无限动画。
  - Edge Function 名称：`generate_animation_config`
  - 输入：`{ type: string, duration: int, easing: string }`
  - 输出：`{ animation_config: object }` 或错误
  - 在 Edge Function 中检查调用者的 JWT 中的 `plan` 是否为 `apex`。
- **客户端调用**：使用 `supabase.functions.invoke('generate_animation_config', payload)`，如果返回 403，提示“需要 APEX 权限”。

#### D2. 应用动画到笔记打开/关闭
- 修改 `NoteEditorActivity` 的启动和退出过渡动画，使用用户保存的动画配置（如果没有自定义，使用默认动画）。
- 具体实现：在 `overridePendingTransition` 或使用 Activity Options Bundle 中根据配置动态设置。

### 模块 E：APEX 功能 - 社区主题分享（上传/下载）

#### E1. 社区主题广场界面 `CommunityThemeActivity`
- **入口**：仅 APEX 用户可见。
- **UI**：两个 Tab：“发现”（所有 `is_public = true` 的主题）和“我的上传”。
- **发现 Tab**：
  - 从 `community_themes` 表查询 `is_public = true`，按 `download_count` 降序排列。
  - 每个主题项显示预览图、名称、作者（可选）、下载次数。
  - 点击“应用”按钮：从 Storage 下载该主题的 `config_url`（需要 APEX 权限，RLS 已限制），应用并提示。
- **我的上传 Tab**：
  - 显示当前用户上传的所有主题（`user_id = 当前用户`）。
  - 每个主题项可编辑、删除、发布/取消发布（切换 `is_public`）。
  - 提供“上传新主题”按钮。

#### E2. 上传新主题流程
- 用户点击“上传新主题”，进入 `UploadThemeActivity`。
- 步骤：
  1. 填写主题名称、描述。
  2. 从本地选择一个已经自定义好的主题配置（即模块 B 中保存的本地主题），或从零开始调整（复用主题编辑器 UI）。
  3. 选择一张预览图（从相册或拍照）。
  4. 点击“上传”。
- 上传逻辑：
  - 先将预览图上传到 `community_theme_assets` bucket，路径格式：`community_theme_assets/{user_id}/{theme_id}/preview.jpg`。
  - 将主题配置 JSON 上传到 `community_theme_assets/{user_id}/{theme_id}/config.json`。
  - 获得两个 URL 后，调用 **Edge Function** `publish_community_theme` 插入记录到 `community_themes` 表。Edge Function 会检查调用者是否为 APEX 用户，并自动填充 `user_id`。
  - 上传成功后，回到“我的上传”列表。

#### E3. 必要的 Edge Function
- **`publish_community_theme`**：
  - 验证 JWT 中的 `plan == 'apex'`。
  - 接收参数：`name`, `description`, `preview_url`, `config_url`。
  - 插入 `community_themes` 记录，`is_public = false`，`download_count = 0`。
- **`delete_community_theme`**：验证用户身份，删除自己上传的主题（同时可删除 Storage 中的文件，可选）。

### 模块 F：APEX 功能 - 云端主题同步（跨设备）

#### F1. 云端同步界面 `CloudThemeActivity`
- **入口**：APEX 用户专属，在设置页显示“我的云主题”。
- **功能**：
  - 列出当前用户保存在 `user_themes` 表中的所有主题（显示名称、更新时间）。
  - 点击某个主题，可以“应用”到本地（下载 `config` 并调用 `ThemeManager.applyTheme`）。
  - 提供“保存当前主题到云端”按钮：将当前本地主题配置（名称可编辑）插入 `user_themes` 表。
- **同步逻辑**：
  - 保存时调用 Edge Function `save_cloud_theme`（检查 APEX 权限）。
  - 删除时调用 `delete_cloud_theme`。
  - 拉取列表直接从 `user_themes` 表查询（RLS 保证只能看到自己的）。

## 四、UI 交互细节要求

- 所有需要权限判断的按钮或入口，在显示前调用 `FeatureGuard.isProOrApex()` 或 `isApex()`（在协程中）。如果权限不足，要么隐藏按钮，要么显示为灰色并点击后弹出升级对话框。
- 所有网络请求（下载主题、调用 Edge Function）必须在后台线程执行，使用 Kotlin Coroutines，并在失败时 Toast 提示用户。
- 主题应用后，建议重启当前 Activity 或通过 `recreate()` 刷新界面。
